题目描述

  1. 对于所有满足 1⩽i⩽n 的 i，令 ci=ai−1×b1+ai×b2+ai+1×b3 ；
  2. 对于所有满足 1⩽i⩽n 的 i，令 ai=ci。

现在珂朵莉想知道，k 次操作后，所有水果个数（ai 的和）是多少。

一行两个整数 n,k。

第二行 n 个整数，表示 ai。

第三行 3 个整数，表示 bi。

输出格式

一行一个整数，表示答案。

输入样例

输出样例

说明

样例解释

第一轮：

a1=4∗5+1∗1+1∗4=25

a2=1∗5+1∗1+4∗4=22

a3=1∗5+4∗1+1∗4=13

第二轮：

a1=13∗5+25∗1+22∗4=178

I. 输在假期

时间限制：1000 ms

空间限制：64 MB

题目背景

不内卷的人毫无前途。

己烷已经放弃了卷生物作业，专心研究 PvZ 理论。

最近他在研究栈位理论，发现可以写外挂，决定将这一理论应用于实践中。

由于实际游戏过于复杂，他将游戏简化成了一个模型并进行改进。

题目描述

PvZ 在一个场地大小为 n×m 的方格图上进行游戏，每个格子中都能种若干个植物。

系统为识别这些植物，给每个植物都分配了一个编号，编号分配方式如下：

  • 游戏开始时，创建一个计数器 cnt=0 和一个栈 s，将 0 压入栈中；
  • 每种下一个植物，从栈顶弹出一个元素作为植物的编号；
  • 每铲除一个植物或有一个植物死亡，将植物编号的值压入栈中；
  • 若有若干个植物同时死亡，按种下植物时的时间早晚依次将它们编号都值压入栈中；
  • 只要栈中没有任何元素，将计数器 cnt 加 1，然后将 cnt 的值压入栈中。

游戏中有一种僵尸叫巨人僵尸，我们用坐标 (x,y) 表示他的位置，他会对植物造成毁灭性打击：

我们记 (x,y) 和 (x,y−1) 中植物的最小编号（没有植物视为无穷大）的值分别为 p 和 q ，若 p⩽q 则位置 (x,y) 的所有植物同时死亡，否则 (x,y−1) 的所有植物同时死亡，随后巨人僵尸死亡。

现在己烷给你若干个操作，让你模拟游戏的运行。

输入格式

第一行 3 个数，分别为 n,m,q。

接下来 q 行，每行开头一个数 op：

  • 若 op=1，接下来输入两个数 x,y,表示在位置 (x,y) 种下一个植物；
  • 若 op=2，接下来输入两个数 x,y,表示在位置 (x,y) 出现一个巨人僵尸；
  • 若 op=3，表示查询栈顶元素，请输出当前栈顶元素。

输出格式

对于每个查询，输出一行一个数 x，表示栈顶元素。

输入样例

输出样例

说明

样例说明

第一次放置植物时，该植物编号为 0；

第二次放置植物时，该植物编号为 1；

第一次查询时，cnt=3；

第三次放置植物时，该植物编号为 2；

巨人出现时，(2,3) 中的植物编号分别为 0,2，(2,4) 中的植物元素编号为 1，巨人杀死了 (2,3) 的所有植物，将 0 和 2 先后压入栈中，此时栈中元素从栈底到栈顶元素分别为 3,0,2；

第二次查询时，栈顶元素为 2。

数据范围

所有数均为正整数。

1⩽n,m⩽103，1⩽q⩽106，1⩽x⩽n，1⩽y⩽m。
